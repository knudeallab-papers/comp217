• 프로젝트 목적
매일매일 경북대학교에서 밥을 먹는 학생들, 밥집을 고르는 게 힘든 사람들, 
무슨 식당이 있는지 모르는 신입생들을 위해 식당을 알려주는 프로그램

• 실행 전 주의사항
1. package 명은 term_project 로 되어있다.
2. 이미지 파일이 들어있는 폴더는 src 폴더 안에 넣어야 정상적으로 이미지가 잘 나온다.
3. 사진이 프로그램에서 정상적으로 뜨지 않는 경우, 각 파일의 위치나 링크를 잘 확인한 후에
다시 실행해보아야 한다.
4. Binary File을 만들고 읽어올 때, 실행결과 각 컴퓨터가 인식을 다르게 하여 오류가 발생한다.
따라서 Review 클래스를 먼저 한 번 실행한 후, mainPage 클래스를 실행하면 오류 없이 잘 작동한다.
5. 혹시 창이 짤리거나 할 시, 컴퓨터마다 픽셀이 다르므로 크기를 수정해보록 한다.

• 프로그램 작동 방법
Review 클래스를 먼저 실행한 후, mainPage 클래스를 실행하면 된다.

• 코드 작동 원리
1. Review 클래스를 실행하면 각 store폴더에 Review 라는 바이너리 파일이 만들어진다.
2. mainPage 클래스를 실행하게 되면 '경북대생을 위한 식당 정보 : 골 목 식 당'이라는 창이 뜨면서
'북문', '정문', '쪽문', '동/텍문', '학식', '뭘 먹을지 고민된다면? 눌러보세요' 라는 6가지의 버튼이 나오게 된다.
3. '뭘 먹을지 고민된다면? 눌러보세요' 라는 버튼을 누르게 되면 recommend 클래스가 생성되면서 
'메뉴를 추천해드려요'라는 창이 뜨면서 날씨를 물어본다. 오늘에 맞는 날씨 버튼을 누르면 밑에 추천하는 메뉴나
추천하는 음식스타일이 나오게 된다.
4. '북문', '정문', '쪽문', '동/텍문', '학식'을 누르게 되면 각 위치에 맞는 RestaurantChoice 클래스가 생성되면서
'각 음식스타일 별로 '한식', '일식', '중식', '양식', '패스트푸드', '기타', '랜덤' 이미지 버튼이 나오게 된다.
5. '한식', '일식', '중식', '양식', '패스트푸드', '기타'를 누르게 되면 음식스타일에 맞는 가게가 이미지와 함께 나오게 된다.
6. 가게 버튼을 누르게 되면 Store 클래스가 형성되면서 가게의 정보를 알려주는 창이 뜨게 된다.
7. 5에서 '랜덤' 이미지 버튼을 누르게 되면 각 위치에 맞는 가게 중에 랜덤하게 
Store 클래스가 형성되면서 하나의 가게의 정보를 알려주는 창이 뜬다.
8. Store 클래스가 형성되면서 Store 폴더에 있는 식당 번호에 맞게끔, 식당에 대한 정보를 받아오고
위에 가게 이름과 함께 'Menu', 'Information', 'Review' 버튼이 나오게 된다. 처음에 창이 열리면
'Menu' 버튼이 눌렸다고 가정하고 각 식당에 있는 메뉴 이미지와 메뉴명, 가격이 보이게 된다.
9. 'Information' 버튼을 누르게 되면 식당의 상호명, 운영시간, 휴무일, 전화번호, 위치가 글로 보인다.
10. 'Review' 버튼을 누르면 가게별로 Review 파일을 읽어와서 얻은 리뷰 정보가 중앙에 나오고, 
위에는 평점의 평균이 나오게 되며, 밑에 별 버튼을 누르고 리뷰를 작성한 후에 'SAVE'버튼을 누르게 되면
Review 클래스를 만들어서 Review 파일에 새로 저장하게 된다.

• 개선하면 좋을 점
1. 모든 가게, 모든 메뉴 정보를 넣지는 못해서 위치에 따른 음식스타일별로 식당 3개 정도만 나와있다.
더 많은 식당 정보를 넣을 수 있으면 좋을 것 같고, 각 이미지 또한 있으면 좋을 것 같다.
2. Review 파일이 바이너리 파일이어서 그런지 다른 컴퓨터에서는 읽어들어오지 못했다. 
3. 처음에는 선호하는 스타일(예를 들어 매운 음식)이 있으면 그와 관련된 가게를 보여주려고 했지만,
구현하지 못해서 이러한 추천 시스템이 있으면 좋을 것  같다.
4. 날마다 바뀌는 학식 같은 경우에는 매일매일 이미지와 메뉴정보를 바꿔줘야 하는 번거로움이 있다.
5. 예약 및 웨이팅 기능, 검색 기능까지 추가하게 되면 더 좋은 프로그램이 될 것 같다.
